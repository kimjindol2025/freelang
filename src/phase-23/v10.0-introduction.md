# v10.0 - Runtime Optimization & Garbage Collection

**프리랭 v10 시대의 개막**

## 🎯 비전: 스스로 진화하는 엔진

v9까지의 ARC가 '개별 객체의 생존 기록'에 집중했다면,
v10은 **엔진 전체의 '생태계 관리'**에 집중합니다.

**엔진이 스스로**
- 자신의 실행을 분석하고 (**Profiling**)
- 자주 실행되는 코드를 가속하며 (**JIT**)
- 메모리 단편화를 정화합니다 (**GC**)

## 🔧 핵심 메커니즘

### 1. Adaptive Optimization (적응형 최적화)

**JIT (Just-In-Time) Compilation**
- Hot-Spot 바이트코드를 감지하여 실시간으로 기계어 변환
- Speculative Execution으로 실행 속도 획기적 향상

**Generational GC**
- ARC만으로 해결하기 어려운 메모리 단편화 정리
- Young/Old 세대별 관리로 GC 중단 시간 최소화

### 2. Code Status Lifecycle

```
COLD (콜드)
  ├─ 거의 실행 안 됨
  ├─ 인터프리터 실행
  └─ 임계치 수집 중

        ↓ (Call Count ≥ 1,000)

WARM (웜)
  ├─ 가끔 실행됨
  ├─ Baseline JIT 준비
  └─ 프로파일 데이터 수집

        ↓ (Call Count ≥ 10,000 + 타입 안정성 감지)

HOT (핫)
  ├─ 자주 실행됨
  ├─ Optimizing JIT 컴파일 시작
  └─ 인라인 캐싱 활성화

        ↓ (Type instability 감지 시)

Deoptimization → WARM/COLD로 복귀
```

### 3. Memory Hierarchy

```
Stack (즉시 해제)
  ↓
TLAB (Thread Local Allocation Buffer)
  ↓
Young Generation (객체 대부분이 여기서 사망)
  ↓ (존속 조건 만족 시)
Old Generation (오래 살아남은 객체들)
  ↓ (GC 마크 단계에서 미도달)
Garbage
```
