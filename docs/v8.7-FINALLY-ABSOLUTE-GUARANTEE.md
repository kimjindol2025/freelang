# FreeLang v8.7: FINALLY - The Absolute Guarantee

## 📌 개요

v8.6까지 예외의 **정체**를 판별했다면, v8.7은 "어떤 상황에서도 실행되어야 하는 정리 코드"를 **절대 보장**하는 단계입니다.

**핵심 원칙**: "약속은 지킨다. FINALLY는 어떤 탈출도 허용하지 않는다."

### Before (v8.6)
```freelang
try {
  // 파일 열기
  openFile("data.txt")

  // 어딘가 RETURN이나 THROW가 발생
  return "Success"

  // 파일 닫기 코드는 실행 안 됨!
  closeFile()
}
catch (e) {
  println("Error: " + e.Message)
}
```

### After (v8.7)
```freelang
try {
  openFile("data.txt")
  return "Success"  // 여기서 나가려고 해도...
}
finally {
  closeFile()  // 이것은 절대 실행된다!
}
```

---

## 🏗️ 아키텍처 설계

### 1. 핵심 메커니즘: 지연된 제어권 (Deferred Control)

**Jump Suspension**:
- TRY/CATCH 블록 내에서 RETURN 또는 THROW가 호출됨
- 엔진은 즉시 점프하지 않음
- 먼저 FINALLY 블록의 주소를 확인하고 그곳으로 이동

**Return Address Stacking**:
- FINALLY로 가기 전, 원래 가려던 목적지를 스택에 백업
- 예시:
  ```
  RETURN "Success" 호출
  → savedReturnValue = "Success"
  → savedReturnAddress = <함수 호출자>
  → PC = FINALLY 블록 주소로 변경
  ```

**Resumption**:
- FINALLY 블록의 마지막 명령 수행
- 백업해둔 주소로 복귀
- 원래 중단된 흐름 재개
  ```
  FINALLY 블록 끝
  → 백업된 "Success" 값으로 RETURN 재실행
  → 호출자에게 반환
  ```

### 2. 엔진 수정 사항

**FinallyContext 저장**:
```typescript
interface FinallyContext {
  pendingReturn?: any;        // 보류된 RETURN 값
  pendingException?: Error;   // 보류된 예외
  returnAddress?: number;     // 복귀할 주소
  hasFinally: boolean;        // FINALLY 블록 존재 여부
}
```

**Control Flow**:
```
TRY 블록 실행 중 RETURN/THROW 발생
  ↓
FinallyContext에 정보 저장
  ↓
PC를 FINALLY 블록 주소로 변경
  ↓
FINALLY 블록 실행
  ↓
FinallyContext의 정보로 원래 제어권 복구
  ↓
RETURN/THROW 재실행 (또는 정상 종료)
```

## 📝 테스트 케이스 상세

### TC_V8_7_001: Return Interception (RETURN 가로채기)

```freelang
let status = "Initial"

fn testReturn() {
  try {
    println("1. Inside Try")
    return "Success"  // 여기서 나가려고 함
  }
  finally {
    status = "Finally Executed"
    println("2. Inside Finally")
  }
}

let result = testReturn()

// 기대: FINALLY가 먼저 실행됨
if (status == "Finally Executed" && result == "Success") {
  println(1)  // ✅ 가로채기 성공
} else {
  println(-1)
}
```

**기대**: 출력 = [1, 2, 3, 1] (순서대로 1→2→Finally→3)

### TC_V8_7_002: Throw Propagation (예외 전파 보존)

```freelang
let cleanup = ""

fn testThrow() {
  try {
    println("1. Inside Try")
    throw new Exception("Error!")  // 예외 발생
  }
  finally {
    cleanup = "Done"
    println("2. Inside Finally")
  }
}

try {
  testThrow()
  println(-1)  // 실행 안 됨
}
catch (e) {
  // 기대: 예외가 여전히 전파됨
  if (cleanup == "Done" && e.Message == "Error!") {
    println(1)  // ✅ 예외 보존 + FINALLY 실행
  }
}
```

**기대**: 출력 = [1, 2, 1] (예외 발생 → FINALLY → 예외 전파)

### TC_V8_7_003: Nested Finally (중첩 FINALLY)

```freelang
let order = ""

fn outer() {
  try {
    println("1. Outer Try")
    try {
      println("2. Inner Try")
      return "Success"
    }
    finally {
      order = order + "Inner"
      println("3. Inner Finally")
    }
  }
  finally {
    order = order + "Outer"
    println("4. Outer Finally")
  }
}

let result = outer()

// 기대: 안쪽부터 차례대로 실행
if (order == "InnerOuter" && result == "Success") {
  println(1)  // ✅ 중첩 순차 실행
} else {
  println(-1)
}
```

**기대**: 출력 = [1, 2, 3, 4, 1] (안쪽 FINALLY → 바깥쪽 FINALLY)

### TC_V8_7_004: Finally Cleanup (자원 정리)

```freelang
let fileHandle = null

fn processFile() {
  try {
    // 파일 열기
    fileHandle = "open"
    println("1. File opened")

    // 중간에 오류 발생
    throw new Exception("Read error")

    println("2. Processing...")  // 실행 안 됨
  }
  catch (e) {
    println("3. Exception caught: " + e.Message)
  }
  finally {
    // 파일 닫기는 반드시 실행
    if (fileHandle != null) {
      fileHandle = null
      println("4. File closed")
    }
  }
}

processFile()

// 기대: FINALLY가 실행되어 fileHandle 정리됨
if (fileHandle == null) {
  println(1)  // ✅ 정리 보장
} else {
  println(-1)
}
```

**기대**: 출력 = [1, 3, 4, 1] (항상 정리됨)

## 🎯 설계 원칙

### "약속의 이행: 엔진은 FINALLY를 거칠 때까지 멈추지 않는다"

```
시나리오 1: RETURN 시도
TRY { return X; FINALLY { cleanup } }
  → cleanup이 먼저 실행됨
  → 그 후 return X 실행

시나리오 2: THROW 시도
TRY { throw E; FINALLY { cleanup } }
  → cleanup이 먼저 실행됨
  → 그 후 throw E 전파

시나리오 3: 정상 종료
TRY { ... FINALLY { cleanup } }
  → 정상 실행 후 cleanup 실행
```

### 핵심: "제어권 추연 (Control Flow Deferment)"

RETURN이나 THROW가 발생해도 **FINALLY 블록이 먼저 실행**됨을 보장합니다.
