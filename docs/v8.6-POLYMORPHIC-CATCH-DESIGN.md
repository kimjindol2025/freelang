# FreeLang v8.6: Polymorphic Catch & Type-based Filtering

## 📌 개요

v8.5까지 예외를 **객체**로 만들었다면, v8.6은 그 객체의 **정체(Type)**를 판별하는 단계입니다.

**핵심 원칙**: "예외의 계급을 이해하고 적절한 바구니에 담기"

### Before (v8.5)
```freelang
try {
  throw new ZeroDivisionError("Cannot divide by zero")
}
catch (e) {
  // 모든 예외를 뭉뚱그려 처리
  println(e.Message)
}
```

### After (v8.6)
```freelang
try {
  throw new ZeroDivisionError("Cannot divide by zero")
}
catch (e: MathError) {
  // MathError와 자식 클래스만 처리
  println("Math problem: " + e.Message)
}
catch (e: Exception) {
  // 나머지 모든 예외 처리
  println("Other error: " + e.Message)
}
```

---

## 🏗️ 아키텍처 설계

### 1. 핵심 메커니즘: vTable 대조

**Type Matching**:
- THROW된 객체의 `__vPtr` (vTable 포인터)가 가리키는 클래스 정보 추출
- CATCH 블록에 명시된 클래스 정보와 대조
- 일치하면 예외를 해당 CATCH 블록에서 처리

**Inheritance Awareness**:
- v7.1 상속 엔진 활용
- ZeroDivisionError가 MathError를 상속하고, MathError가 Exception을 상속하면
- ZeroDivisionError 예외는 MathError CATCH에서도 잡을 수 있음
- 부모 클래스 체인을 따라가며 매칭 검사

**Fallthrough**:
- 현재 CATCH 블록의 타입과 맞지 않으면 다음 CATCH 블록 검사
- 모든 CATCH가 실패하면 상위 핸들러로 예외 전파 (재던지기)

### 2. 엔진 수정 사항

**Multi-Catch Block Support**:
```typescript
interface TryStatement extends ASTNode {
  type: 'TryStatement';
  tryBlock: BlockStatement;
  catchBlocks: CatchBlock[];  // 여러 CATCH 블록
  finallyBlock?: BlockStatement;
}

interface CatchBlock extends ASTNode {
  type: 'CatchBlock';
  exceptionType?: string;     // CATCH의 타입 필터 ("MathError" 등)
  exceptionVar: string;       // 예외 변수 이름
  body: BlockStatement;
}
```

**Type Filtering Logic**:
1. THROW 발생 → exceptionObject 저장 (v8.5)
2. 각 CATCH 블록 순회
3. 각 CATCH마다:
   - exceptionType이 없으면 → 모든 예외 수용 (v8.5 호환)
   - exceptionType이 있으면 → 타입 매칭 검사
4. 매칭된 CATCH에서 처리, 또는 다음 CATCH로 진행

**isInstanceOf(exceptionObject, expectedType)**:
```typescript
private isInstanceOf(obj: any, expectedType: string): boolean {
  // 1. 정확한 타입 일치
  if (obj.__class === expectedType) {
    return true;
  }

  // 2. 상속 관계 확인 (v7.1)
  const classDef = this.classTable.get(obj.__class);
  let current = classDef;
  while (current) {
    if (current.name === expectedType) {
      return true;
    }
    // 부모 클래스로 이동
    current = current.superClass ? this.classTable.get(current.superClass) : null;
  }

  return false;
}
```

## 📝 테스트 케이스 상세

### TC_V8_6_001: Exact Match (정확한 타입 일치)

```freelang
class MathError extends Exception {}
class ZeroDivisionError extends MathError {}

try {
  throw new ZeroDivisionError("Cannot divide by zero")
}
catch (e: ZeroDivisionError) {
  // 정확한 타입 일치 → 즉시 캐치
  if (e.Message == "Cannot divide by zero") {
    println(1)  // ✅
  }
}
catch (e: Exception) {
  // 실행 안 됨 (이미 위에서 잡혔음)
  println(-1)
}
```

**기대**: 출력 = [1] (정밀 매칭 성공)

### TC_V8_6_002: Parent Match (부모 타입 매칭)

```freelang
class MathError extends Exception {}
class ZeroDivisionError extends MathError {}

try {
  throw new ZeroDivisionError("Cannot divide by zero")
}
catch (e: TypeError) {
  // 타입 불일치 → 스킵
  println(-1)
}
catch (e: MathError) {
  // 부모 타입 매칭 → 수용!
  if (e.Message == "Cannot divide by zero") {
    println(1)  // ✅
  }
}
catch (e: Exception) {
  // 실행 안 됨
  println(-1)
}
```

**기대**: 출력 = [1] (상속 필터링 성공)

### TC_V8_6_003: Unmatched Skip (타입 불일치 시 스킵)

```freelang
class NetworkError extends Exception {}
class ValidationError extends Exception {}

try {
  throw new NetworkError("Connection timeout")
}
catch (e: ValidationError) {
  // 타입 불일치 → 스킵
  println(-1)
}
catch (e: Exception) {
  // Exception으로 수용 (폴백)
  if (e.Message == "Connection timeout") {
    println(1)  // ✅
  }
}
```

**기대**: 출력 = [1] (폴백 처리 성공)

### TC_V8_6_004: v8.5 Compatibility (타입 없는 CATCH)

```freelang
class ZeroDivisionError extends Exception {}

try {
  throw new ZeroDivisionError("Cannot divide by zero")
}
catch (e) {
  // v8.5 호환: 타입 필터 없으면 모든 예외 수용
  if (e.Message == "Cannot divide by zero") {
    println(1)  // ✅
  }
}
```

**기대**: 출력 = [1] (하위 호환성 유지)

## 🎯 설계 원칙

### "예외의 계급을 이해하고 적절한 바구니에 담기"

```
Exception (최상위)
  ├─ MathError
  │   └─ ZeroDivisionError ← THROW
  │
  └─ NetworkError
      └─ ConnectionTimeoutError

CATCH 블록 순회:
1. catch (e: TypeError) → 불일치 (ZeroDivisionError ≠ TypeError) → 스킵
2. catch (e: MathError) → 일치! (ZeroDivisionError는 MathError의 자식) → 처리 ✅
3. catch (e: Exception) → 이미 처리됨 → 스킵
```

### 핵심: "상속 관계를 따라가며 검사하기"

```typescript
ZeroDivisionError → MathError → Exception → 찾음!
```

```freelang
catch (e: MathError) {
  println("Math error: " + e.Message)
}
finally {
  // 예외 발생 여부와 관계없이 항상 실행
  println("Cleanup code")
}
```
