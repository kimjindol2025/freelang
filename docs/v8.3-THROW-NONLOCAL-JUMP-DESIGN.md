# FreeLang v8.3: THROW & Non-local Jump (The Great Escape)

## 📌 개요

v8.1에서 "어디로 튈 것인가"(핸들러 위치)를 정했고,
v8.2에서 "어떤 상태로 튈 것인가"(스냅샷)를 저장했다면,
v8.3은 **"지금 당장 튈 것인가"**(실제 점프 실행)를 구현하는 단계입니다.

**핵심 원칙**: 기록되지 않은 곳으로는 점프할 수 없다.
- 핸들러 스택이 비어있으면 엔진은 패닉 (안전장치)
- PC를 강제로 변경하여 CATCH 블록으로 점프

---

## 🏗️ 아키텍처 설계

### 1. THROW 메커니즘 (3단계)

```
① Immediate Interruption
   ↓ 현재 함수 실행 중단 (나머지 코드 무시)

② Handler Lookup
   ↓ handlerStack 최상단 확인
   ↓ v8.2 스냅샷에서 복구 PC 읽기

③ Instruction Pointer Reset
   ↓ this.pc를 핸들러 PC로 덮어씌움
   ↓ 다음 명령어가 CATCH 블록이 됨
```

### 2. Non-local Jump vs Local Jump

```
Local Jump (일반 JMP):
  현재 함수 내에서만 점프
  예: if (cond) { JUMP to_label }

Non-local Jump (THROW):
  함수 경계를 허물고 상위 호출자로 점프
  예: deep_function() → middle_function() → THROW
       → 즉시 try-catch 블록으로 점프
```

### 3. PC 변경 시점

```
Before THROW:
  this.pc = 10 (현재 실행 중인 명령)
  나머지 코드 (명령 11, 12, 13...) 준비됨

THROW 실행:
  ① handlerStack 조회
  ② savedPC 읽기 (예: 50 = CATCH 블록 시작)
  ③ this.pc = 50으로 변경
  ④ exception 객체 기록

After THROW:
  명령 11, 12, 13... 완전히 무시됨
  다음 명령어: 50번 (CATCH 블록)
```

## 📝 테스트 케이스 상세

### TC_V8_3_001: PC Redirect (PC 재지정)

**목적**: THROW 호출 시 PC가 정확히 핸들러 PC로 변경되는가?

```freelang
let checkpoint = 0

try {
  println(1)
  throw new Exception("Jump!")

  // 이 코드는 실행되면 안 됨
  checkpoint = 1
  println(999)
}
catch (e) {
  // PC가 여기로 점프해야 함
  println(2)
}

// checkpoint는 여전히 0 (건너뜀 증명)
if (checkpoint == 0) {
  println(3)
} else {
  println(-1)
}
```

**기대 흐름**:
```
println(1) → 1 출력
throw 실행 → PC 변경 (CATCH로 점프)
checkpoint = 1, println(999) → 건너뜀 ✅
CATCH 블록 진입
println(2) → 2 출력
checkpoint 체크 → 0 (변경 안 됨)
println(3) → 3 출력

최종 출력: [1, 2, 3]
```

### TC_V8_3_002: Bypass Logic (코드 건너뜀)

**목적**: THROW 아래의 여러 명령문들이 모두 무시되는가?

```freelang
let x = 0, y = 0, z = 0

try {
  x = 10
  throw new Exception("Skip all")

  // 다음 6개 명령 모두 무시되어야 함
  y = 20
  z = 30
  println(999)
  x = 999
  y = 999
  z = 999
}
catch (e) {
  // PC 점프로 여기 도착
}

// 검증: x=10, y=0, z=0 (할당 전 값)
if (x == 10 && y == 0 && z == 0) {
  println(1)
} else {
  println(-1)
}
```

**기대 결과**:
```
x = 10 (할당됨)
throw 실행 → PC 변경
y = 20, z = 30, ... 모두 건너뜀 ✅
x == 10, y == 0, z == 0 확인 ✅
println(1) → 1 출력
```

### TC_V8_3_003: Safety Landing (안전 착륙)

**목적**: 점프한 CATCH 블록이 유효하고 정상 실행되는가?

```freelang
let result = []

try {
  println(1)
  result.push(1)

  throw new Exception("Landing test")

  result.push(999)
}
catch (e) {
  // PC 점프로 여기 착륙
  result.push(2)
  println(2)
}

result.push(3)
println(3)

// 기대: [1, 2, 3] (999 없음)
if (result.length == 3 && result[0] == 1 && result[1] == 2 && result[2] == 3) {
  println(4)
}
```

**기대 결과**:
```
result = []
result.push(1) → [1]
throw 실행 → PC 변경
result.push(999) → 건너뜀
result.push(2) → [1, 2]
result.push(3) → [1, 2, 3]
검증 통과 ✅
println(4) → 4 출력

최종 출력: [1, 2, 3, 4]
```

## 🚨 안전장치: PANIC_NO_HANDLER

```typescript
case 'ThrowStatement': {
  // THROW 실행 전 핸들러 확인
  if (this.handlerStack.length === 0) {
    this.log(`[PANIC_NO_HANDLER] throw 실행 중 핸들러 없음!`);
    throw new Error('[PANIC] No active handler for throw');
  }

  // 정상 경로: PC 변경
  const handler = this.handlerStack[this.handlerStack.length - 1];
  const jumpPC = handler.snapshot?.savedPC;
  this.pc = jumpPC; // ← PC 강제 변경
  throw new Error(message); // ← 외부 catch로 명령어 완전 중단
}
```

## 🎯 설계 원칙

### "기록이 증명이다" in v8.3

```
기록되지 않은 곳으로는 점프할 수 없다.

v8.1에서 핸들러 등록 ← 기록
v8.2에서 PC 저장 ← 증명
v8.3에서 PC로 점프 ← 실행

기록 없음 = 점프 불가능 = 패닉!
```

### Non-local Jump의 의미

```
일반 함수 호출 (Local):
  caller → callee → return to caller

예외 처리 (Non-local):
  deep_function
    ↓ calls
  middle_function
    ↓ calls
  throw_function
    ↓ THROW (기록된 위치로 직접 점프)
  try-catch block ← 여기로 점프!
```
